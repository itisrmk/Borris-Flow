# About Borris-Flow

**Borris-Flow** is a production-oriented starter that standardizes Claude Code usage across projects.

### Inspiration (credit)

Boris Cherny’s public thread (https://x.com/bcherny/status/2007179832300581177) inspired this project’s workflow approach.
We replicated only the process ideas (create/execute/review handoffs and local conventions), not implementation code.

This repo is built as an independent, workflow-first scaffold.

### Goals

- Make Claude first-class in your local workflow.
- Reduce setup friction when starting a new repository.
- Enable fast parallel feature execution without sacrificing clarity.
- Keep local automation safe, predictable, and non-destructive.

### Who it is for

- Engineers building with Claude Code frequently.
- Teams wanting repeatable conventions for command/agent workflows.
- Builders who prefer scripts that are easy to audit, easy to fork, and easy to run.

### In short

Install once, initialize projects quickly, and run multiple feature branches from a single command.
